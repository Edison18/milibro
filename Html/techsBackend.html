TECNOLOGÍAS BACKEND

Como mencionamos anteriormente frontend y backend son sistemas de software que interáctuan entre sí, pero mantienen cierta independencia el uno del otro. En este caso en el backend tenemos las aplicaciones del lado del servidor donde se almacenas los datos y el fronted accede a esta mediante consultar que el cliente realiza.

También se le puede conocer como encargado de las opciones CRUD (Create, Read, Update and Delete), también conocidas con el sinónimo de gestionar, en este los datos, la información.

De hecho, las tecnologías backend bien elegidas pueden garantizar la escalabilidad, la velocidad de funcionamiento y responder instantáneamente a las consultas y necesidades de los clientes. Sin embargo, elegir la opción adecuada es un desafío, especialmente si no es programador. Normalmente y desde hace tiempo el lenguje utilizado en el backend es distinto al frontend, esto porque antes no existia forma de usar el mismo lenguaje. A día de hoy mediante servidores se puede programar el código del frontend y backend en el mismo lenguaje, aunque esta práctica se está usando mucho aún se sigue teniendo en cuenta realizarlo con diferentes lenguajes por temas de rendimiento, especilización, escabilidad, rendimiento y ser multiplataforma.

Ejemplo de lo mencionado anteriormente tenemos a JavaScript con el servidor nodeJs, el cual le permite interoperar en el lado del cliente y en el lado del servidor.
